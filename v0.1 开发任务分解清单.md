## v0.1 开发任务分解清单（可直接进 Jira/禅道）

下面按“模块 -> 任务 -> 产出 -> 验收标准（DoD）”拆分，默认 8 周、1-2 名开发可执行。

---

## 0. 项目初始化（第 1 周）

### TASK-0001 解决方案结构搭建
- **内容**：创建项目结构：`FlowEngine`、`ActionSdk`、`ActionBuiltin`、`ScriptHost`、`Persistence`、`FlowDesigner(WPF)`、`Tests`
- **产出**：可编译的多项目解决方案
- **DoD**：
  - `dotnet build` 通过
  - 各模块有基础接口与 README

### TASK-0002 基础工程规范
- **内容**：日志框架、异常中间层、配置加载、统一返回模型
- **产出**：`ILogger` 接入、`appsettings`、全局异常处理
- **DoD**：
  - 所有模块可输出结构化日志
  - 未捕获异常有统一错误码

---

## 1. 流程 DSL 与模型（第 1-2 周）

### TASK-1001 流程 JSON Schema 定义
- **内容**：定义 `flowId/name/version/variables/steps` 结构
- **产出**：`flow.schema.json`
- **DoD**：
  - 示例流程可通过 schema 校验
  - 非法字段会报可读错误

### TASK-1002 核心实体建模
- **内容**：`FlowDefinition`、`FlowStep`、`ExecutionContext`、`StepExecutionResult`
- **产出**：领域模型与序列化配置
- **DoD**：
  - JSON 与对象双向转换通过
  - 覆盖单测（正常/空值/异常）

### TASK-1003 变量与绑定表达式（基础版）
- **内容**：支持 `{{VarName}}` 解析、输入常量/变量映射
- **产出**：`IVariableResolver`
- **DoD**：
  - 至少支持字符串、数字、对象映射
  - 变量不存在时报明确错误

---

## 2. 执行引擎（第 3-4 周）

### TASK-2001 顺序执行器
- **内容**：按步骤顺序执行 `CallMethod/RunScript/Delay/Assert`
- **产出**：`FlowRunner`
- **DoD**：
  - 支持运行、停止
  - 每步有状态与耗时记录

### TASK-2002 控制流支持
- **内容**：`If`、`ForEach`、`TryCatch`
- **产出**：控制流节点执行器
- **DoD**：
  - 条件分支正确跳转
  - 循环变量可传入子步骤
  - 异常可被 Catch 捕获并继续

### TASK-2003 超时与重试机制
- **内容**：步骤级 `timeoutMs`、`retry`、`onError`
- **产出**：统一策略执行器
- **DoD**：
  - 超时后可中断当前步骤
  - 重试次数准确且有日志
  - `Stop/Continue` 策略生效

---

## 3. Action SDK + C# 动作（第 5 周）

### TASK-3001 动作接口与元数据
- **内容**：实现 `IActionHandler`、`ActionMetadata`、参数定义模型
- **产出**：`ActionSdk` v0.1
- **DoD**：
  - 动作可声明输入/输出/说明
  - 引擎按名称找到动作并执行

### TASK-3002 动作注册机制（反射）
- **内容**：扫描带特性类，注册到容器
- **产出**：`ActionRegistry`
- **DoD**：
  - 新增动作无需改引擎代码
  - 重名动作有冲突提示

### TASK-3003 内置动作首批（>=5）
- **内容**：`Delay`、`SetVariable`、`HttpRequest`、`ReadFile`、`AssertEquals`
- **产出**：`ActionBuiltin`
- **DoD**：
  - 每个动作有单测
  - 可在流程中串联执行并传值

---

## 4. ScriptHost（第 6 周）

### TASK-4001 脚本运行器（单语言）
- **内容**：建议先接 `PowerShell`（Windows 场景）
- **产出**：`RunScript` 动作
- **DoD**：
  - 输入变量可注入脚本
  - 脚本输出可回写上下文

### TASK-4002 脚本隔离与安全
- **内容**：运行超时、工作目录限制、敏感参数脱敏日志
- **产出**：脚本执行策略
- **DoD**：
  - 恶性脚本可被超时终止
  - 日志不打印明文密码

---

## 5. 持久化与运行记录（第 6-7 周）

### TASK-5001 流程保存/加载
- **内容**：JSON 文件读写 + schema 校验
- **产出**：`IFlowRepository`
- **DoD**：
  - 支持新建、保存、另存为、加载
  - 加载失败提示具体字段问题

### TASK-5002 执行记录落盘
- **内容**：SQLite 存 `RunRecord/StepRecord`
- **产出**：运行历史查询接口
- **DoD**：
  - 可按流程名/时间查询历史
  - 历史详情可查看每步结果

---

## 6. FlowDesigner UI（第 7 周）

### TASK-6001 流程编辑页
- **内容**：步骤列表、增删改、上移下移、属性面板
- **产出**：主编辑界面（WPF + MVVM）
- **DoD**：
  - 编辑操作实时反映到模型
  - 可保存并重新打开一致

### TASK-6002 动作选择与参数配置页
- **内容**：动作检索、参数表单、输入绑定（常量/变量）
- **产出**：动作配置 UI
- **DoD**：
  - 参数类型校验（必填/类型）
  - 输出变量映射可配置

### TASK-6003 运行控制台
- **内容**：运行、停止、单步；步骤日志实时展示
- **产出**：运行监控面板
- **DoD**：
  - 能看到当前执行到哪一步
  - 失败时展示错误与堆栈摘要

---

## 7. 测试与验收（第 8 周）

### TASK-7001 单元测试
- **内容**：核心模块覆盖：DSL、引擎、动作、脚本、持久化
- **产出**：`Tests` 项目
- **DoD**：
  - 单测 >= 20
  - 核心路径覆盖率建议 >= 70%

### TASK-7002 集成测试（端到端流程）
- **内容**：准备 5 条示例流程并自动跑通
- **产出**：E2E 测试脚本与报告
- **DoD**：
  - `Happy path` + `Fail path` 均覆盖
  - 报告可定位失败步骤

### TASK-7003 MVP 发布包
- **内容**：打包、版本号、发布说明、已知问题
- **产出**：`v0.1` 安装包/可执行包
- **DoD**：
  - 新机器按说明可运行
  - 至少 1 名试用者可完成完整流程操作

---

## 建议的 Jira 字段模板（可复制）

- `标题`：`[模块] 动词 + 对象`（例：`[FlowEngine] 实现步骤超时与重试`）
- `类型`：Epic / Story / Task / Bug
- `优先级`：P0/P1/P2
- `前置依赖`：任务编号
- `验收标准`：3-5 条可测试语句
- `工时`：1d / 2d / 3d / 5d
- `输出物`：代码路径 + 文档路径 + 测试项
